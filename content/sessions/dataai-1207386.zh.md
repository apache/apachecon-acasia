AI Agent 在生产中失败，往往不是因为模型不够强，而是因为底下的数据栈太弱。割裂的数据孤岛、口径不一的定义，以及埋在"部落知识"里的业务逻辑，让 Agent 陷入数据过载、却对上下文一无所知的困境。

Apache Iceberg 修复了地基。ACID 事务、时间旅行与模式演进，把存储变成一个 Agent 可以可靠地进行推理的、活的、带版本的上下文层。dbt 则修复了"语义"。语义模型与 dbt MCP 服务器，为 Agent 提供了一个受治理的数据接口——把自然语言翻译成扎根于业务逻辑的结构化查询。

二者结合，便构成了 Structured RAG：一种理解模式、尊重治理、并返回可解释结果的检索方式。

本次演讲将包含一个完全基于开源、构建在 Iceberg 与 dbt 之上的 Structured RAG 技术栈的现场 Demo，涵盖语义查询转换、混合检索，以及通过 dbt MCP 实现的受治理的 Agent 推理。可以期待架构图、真实代码与实操指南。
